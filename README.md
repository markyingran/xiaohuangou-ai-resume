## 小黄狗 AI 改简历（前端 Demo）

本仓库实现了 Web 版「小黄狗 AI 改简历」的前端交互 Demo，对应产品方案中的 A/B/C 三个区域布局与主要交互流程。

### 核心用户路径

- **主路径：有旧简历的用户**
  1. 在 A 区「招聘需求」粘贴目标岗位 JD。
  2. 在 A 区切到「上传之前简历」，上传自己的 PDF/Word 简历。
  3. 可选：在 B 区和 AI 聊天，补充自己的职业经历、偏好等。
  4. 满足「有 JD + 有个人信息来源」后，C 区底部「生成简历」按钮视觉点亮。
  5. 点击「生成简历」，在 C 区生成一版简历预览，并创建一个带有日期和序号的版本 Tab（例如 `3月-2日-1`）。
  6. 如果不满意，在 B 区继续提出修改建议，再次点击「生成简历」，得到新版本（如 `3月-2日-2`），可以在 Tabs 中自由切换历史版本。
  7. 确认满意后，点击「导出简历」按钮（当前 Demo 用弹窗提示代替真实导出能力）。

- **备选路径：没有旧简历的用户**
  1. 在 A 区「招聘需求」粘贴 JD。
  2. 不上传旧简历，直接在 B 区聊天里，详细描述自己的工作经历、项目成果、教育与技能等。
  3. 当系统检测到 B 区已经有用户的经历描述后，「生成简历」按钮同样会点亮。
  4. 后续生成简历、反复优化、切换版本和导出流程与主路径一致。

### 按钮与状态说明

- **生成简历**
  - 当 JD 为空时：按钮置灰，点击会提示「请填入对方的招聘需求」。
  - 当 JD 不为空、但既没有上传简历，也没有在 B 区提供经历时：按钮置灰，点击提示「请上传自己之前的简历，或者跟我详细聊聊你的职业经历」。
  - 当 JD 不为空，且满足「上传过简历」或「在 B 区说过自己的经历」任一条件时：按钮变为绿色点亮，点击会生成新的简历版本 Tab，并将其展示在 C 区。

- **导出简历**
  - 只有在已经存在至少一个简历版本的情况下才有实际意义；如果没有任何版本，点击会提示「请先生成一份简历」。

- **重新来**
  - 右上角全局按钮，点击会弹出确认框。
  - 选择「确认」会清空所有输入、聊天记录和简历版本，将页面恢复到初始状态；选择「取消」则不做任何改动。
