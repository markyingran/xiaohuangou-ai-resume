# 小黄狗 · 改简历 — 阶段性项目小结

> 用于保留项目现状与关键决策，便于后续上下文不足时快速恢复共识。最后更新：按当前代码与对话整理。

---

## 一、项目当前形态（对外可怎么描述）

- **产品定位**：  
  「小黄狗 · 改简历」——帮候选人基于目标岗位 JD，对简历做**定向润色与排版**，突出匹配点，同时**显式标记 AI 新增内容，避免造假**。
- **核心用户流程（MVP 已具备）**：  
  1. 输入/粘贴招聘 JD。  
  2. 上传自己的简历（PDF/Word 自动解析为文本），或在对话区补充经历。  
  3. 填写邀请码（控制体验次数），进入应用。  
  4. 点击「生成简历（剩余 x 次）」生成新版本，C 区展示，支持版本切换与预览。  
  5. 可导出为 Word 文档，用于正式投递。

---

## 二、已经实现的主要功能模块

### 前端（Next.js + React + Tailwind）

- **布局**
  - 左侧 C 区：简历预览 + 版本管理（按日期与生成次数排序，最新在前）。
  - 右上 A 区：Tab 1 招聘需求（JD 输入 + 保存）；Tab 2 上传之前的简历（PDF/DOCX 上传、解析、替换文件）。
  - 右下 B 区：用户和 AI 的聊天区域，用于补充经历/偏好。
- **按钮逻辑**
  - JD 为空 → 生成按钮置灰，提示「请填入对方的招聘需求」。
  - JD 有内容但既无上传简历又无对话 → 仍置灰，提示「请上传简历或详细聊聊经历」。
  - 有 JD 且有简历或聊天内容 → 绿色点亮，可生成。
- **多版本管理**：每次成功生成一版创建一个 tab，命名「x月-x日-第n次」，支持切换查看历史版本。
- **全局重置**：「重新来」按钮 + 确认弹窗，一键清空 JD / 上传 / 对话 / 版本。
- **邀请码门禁**：进入页先弹出毛玻璃遮罩 + 邀请码输入；校验 `NEXT_PUBLIC_INVITE_CODES` 与剩余次数；生成消耗次数，localStorage 记录；顶部「退出登录」清空邀请码状态，需重新输入有效码才可继续。
- **导出简历**：使用 `docx` 在前端生成 Word；导出前去掉 `[[AI新增]]` 标记，仅保留纯文本。

### 后端 API

- **`/api/upload-resume`**：接收 PDF/DOCX，用 `pdf-parse` / `mammoth` 提取纯文本并返回。
- **`/api/generate-resume`**：对接阿里云 DashScope（Qwen Plus）OpenAI 兼容接口；无 API Key 时返回 Demo 文本；返回 `resumeText`（带 AI 标记）、`summary`、`usedSignals`。

## 三、AI 生成逻辑：从单阶段到两阶段流水线

### 早期版本（已淘汰）

- 把 JD + 原简历 + 对话一次性给模型，直接输出完整简历；system prompt 要求「不得造假」「用 [[AI新增]] 标记新增」。
- 问题：模型易造假/脑补、标黄不稳定、删真实内容、把姓名/电话/公司名脱敏成 XX。

### 当前版本：两阶段生成 + 强约束

- **阶段一：事实抽取（Fact Extraction）**
  - `extractFacts`：从「原简历 + 对话」抽取用户真实写过/说过的事实（公司、职位、时间、已有数字等），严禁从 JD 编经历；输出 JSON `{ facts: Fact[] }`；失败则回退为「无事实列表」。
- **阶段二：基于事实重写（Rewrite with Facts）**
  - `generateResumeWithFacts`：输入 JD + facts + 原简历全文 + 上一版简历（可选）。
  - 约束要点：尽量覆盖 facts 重要事实不随意删；只能润色不能凭空新增经历；不在 facts/原简历中的内容一律 `[[AI新增]]...[[/AI新增]]`；原简历中的姓名、邮箱、电话、公司名必须原样保留不得替换为 XX。
- **前端**：`formatResumeContent` 将 `[[AI新增]]...[[/AI新增]]` 渲染为黄色高亮；顶部有提示说明黄色为 AI 补充需用户核查。

## 四、已知的产品与技术取舍

- **产品**：优先保证端到端可用 + 防止明显造假；AI 定位为「在真实事实基础上排版+润色+对齐 JD，并明确标出 AI 补充」。
- **技术**：默认对接 DashScope Qwen；两阶段流水线 + prompt 设计 + 标记机制作为可迁移中间层；保留 Demo 模式；邀请码为 `NEXT_PUBLIC_INVITE_CODES` + localStorage 轻量实现，未接后端计费。
